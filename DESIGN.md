# Design Philosophy

## 解きたい問題

LMベースのASR（Whisper等）は強力だが、構造的にハルシネーションする。

- 未知語・新語・固有名詞を「知っている単語」に引き寄せて誤認識する
- 無音区間にテキストを捏造することがある
- これはデータを増やしても治らない。言語モデルが強いことそのものが原因

リアルタイム音声対話では、この性質は致命的。間違いは downstream で補正できるが、ハルシネーションは検出すら難しい。

## アプローチ

**音響特徴だけで、聞こえた音をかな/音素に忠実に写し取る。**

```
音声 → wav2vec2 encoder → かな (CTC) + 音素 (InterCTC)
```

- 言語モデルを持たない。「それっぽい文」を生成する能力を意図的に持たせない
- 出力はひらがな列と音素列。漢字変換・意味理解は downstream に委ねる
- 誤りは「音響的に近い音への誤認識」に限定される。語彙に存在しない単語を捏造しない

## トレードオフ

| | LMベースASR (Whisper等) | このモデル (Acoustic CTC) |
|---|---|---|
| 既知語の精度 | 非常に高い | 低い |
| 未知語の扱い | ハルシネーション (既知語に変換) | 音響的に忠実 (間違いはあるが嘘はない) |
| 出力形式 | 漢字かな混じり | ひらがな + 音素 |
| レイテンシ | 数百ms〜数秒 | ~100ms |
| 用途 | 書き起こし、要約 | リアルタイム対話、意図検出 |

精度とハルシネーション耐性はトレードオフの関係にある。このモデルは精度を犠牲にしてハルシネーション耐性を取る。

## Dual CTC 設計

Apple "Diverse Modeling Units" (Interspeech 2024) を日本語に適用:

- **Layer 12 → かな CTC (primary)**: 最終出力。対話システムへの入力
- **Layer 6 → 音素 InterCTC (auxiliary)**: 訓練時の正則化 + VSR (読唇) との共有表現

音素出力は語彙に依存しない最小単位なので、未知語に対して最もバイアスが少ない。音素精度が上がれば、音素→かなの規則変換でかな精度も向上する (LM不要)。

## 精度改善の方針

LMを足すとハルシネーション耐性を失う。音響側だけで精度を上げる:

1. **データ量**: 100h → 1,000h → 19,000h (ReazonSpeech)
2. **エンコーダ大型化**: wav2vec2-base (95M) → large (300M)
3. **CTC → RNN-T (Transducer)**: prediction network はかな遷移確率 (bigram/trigram程度) のみ学習。語彙知識ではなく音韻論的制約なので、ハルシネーションリスクは極めて低い
4. **InterCTC最適化**: 音素精度を先に上げて、かなに波及させる

RNN-Tは「LMを足す」のとは本質的に異なる。「きょ」の後に「う」が来やすいという音韻パターンは学ぶが、「渋谷」を「渋谷区神南」にハルシネーションする能力は持たない。

## 推論ターゲット

- **学習**: A100 / Vast.ai (1,000h: ~$300-400)
- **推論**: M2 MacBook Air 16GB, 100ms以内
- wav2vec2-large (300M) でも M2 推論は余裕 (~50-150ms/発話)

## 現状 (2026-02-22)

- wav2vec2-base + 100h (ReazonSpeech small): **KER 34%, PER 31%**
- ただしこの数値はデータリークを含む楽観値:
  - wav2vec2-base の事前学習データ (ReazonSpeech v2.0) に評価音声が含まれる
  - fine-tune の train split (95%) が評価対象に含まれている
- 長い発話 (8-15s) のほうが短い発話 (< 3s) より精度が高い (self-attention の文脈効果)
- 長音 (ー) の KER 87% が最大の弱点
- 短く区切っても精度は上がらない (文脈が減るため)
